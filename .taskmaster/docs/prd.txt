# 견적서 관리 시스템 PRD (Product Requirements Document)

## 프로젝트 개요
motionsense.co.kr 도메인 사용자들을 위한 견적서 관리 시스템 개발

## 현재 구현 완료된 기능
1. Google OAuth 인증 시스템 (@motionsense.co.kr 도메인 제한)
2. 사용자 관리 시스템 (super_admin 권한)
3. 권한별 기능 접근 제어 시스템
4. 대시보드 레이아웃 및 네비게이션

## 현재 해결 중인 문제
1. OAuth 무한 로딩 문제 (진행 중)
2. 사용자 추가 후 권한 부여 기능 완성

## 향후 구현 필요한 기능

### 핵심 비즈니스 기능
1. 견적서 생성 및 관리
   - 견적서 작성 폼
   - 견적서 목록 조회
   - 견적서 수정/삭제
   - 견적서 승인 워크플로우
   - PDF 내보내기

2. 고객 관리
   - 고객사 등록
   - 고객사 정보 관리
   - 고객사별 견적서 이력

3. 공급업체 관리
   - 공급업체 등록
   - 공급업체 정보 관리
   - 공급업체별 품목 관리

4. 품목 관리
   - 품목 등록 및 카테고리 분류
   - 품목별 가격 관리
   - 재고 관리
   - SKU 관리

### 부가 기능
5. 프로젝트 관리
   - 프로젝트별 견적서 그룹핑
   - 프로젝트 진행 상황 추적

6. 정산 관리
   - 매출 현황 보고서
   - 견적서별 수익성 분석

7. 알림 시스템
   - 견적서 승인 알림
   - 기한 임박 알림
   - 시스템 공지사항

8. 시스템 관리
   - 사용자 권한 관리 (완료)
   - 시스템 설정
   - 백업 및 복구

## 기술 스택
- Frontend: Next.js 14 (App Router), Material-UI, TypeScript
- Backend: Supabase (PostgreSQL, Auth, RLS)
- Deployment: Vercel
- Authentication: Google OAuth with domain restriction

## 권한 체계
- super_admin: 모든 기능 접근 가능 (lewis@motionsense.co.kr)
- admin: 사용자 관리 제외한 모든 기능
- member: 기본 견적서 작성 및 조회

## 우선순위
1. 높음: 견적서 CRUD 기능
2. 높음: 고객/공급업체/품목 관리
3. 중간: 프로젝트 관리 및 정산
4. 낮음: 고급 보고서 및 분석

## 성공 기준
- 사용자가 견적서를 생성하고 PDF로 내보낼 수 있음
- 관리자가 사용자 권한을 체계적으로 관리할 수 있음
- 모든 데이터가 안전하게 보호됨 (RLS 적용)
- 반응형 디자인으로 모바일에서도 사용 가능
